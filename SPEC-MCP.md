# SPEC-MCP-001: fmo MCP Server

## Objective

Add an MCP (Model Context Protocol) server to the claris-odata-cli package so that Claude Desktop and other MCP clients can access the full CLI from sandboxed environments. The MCP server acts as a thin shell-execution bridge ‚Äî it does not duplicate CLI logic. As the CLI grows, the MCP server grows automatically.

**Target users:** Claude Desktop users who have claris-odata-cli installed and want to query FileMaker OData APIs, manage credentials/servers, and run scripts directly from Claude's conversation interface.

---

## Commands / Tools Exposed

### `fmo_help`

Get the CLI help text for any command.

| Field | Value |
|-------|-------|
| Input | `{ command?: string }` ‚Äî optional subcommand string, e.g. `"list"` or `"credentials add"` |
| Execution | Runs `node dist/index.js --help` or `node dist/index.js <cmd...> --help` |
| Output | Help text as a single text content block |
| Timeout | 10 seconds (local process, no network) |
| Error handling | Non-zero exit ‚Üí return stderr as text (not `isError`; help failures are non-fatal) |

### `fmo_run`

Execute any fmo CLI command.

| Field | Value |
|-------|-------|
| Input | `{ command: string }` ‚Äî full argument string after `fmo`, e.g. `"list Contacts --server myserver --json"` |
| Execution | Runs `node dist/index.js <args...>` via `execFile` |
| Output | stdout as text; stderr appended under `--- stderr ---` separator if non-empty |
| Timeout | 90 seconds (covers FileMaker server latency, script execution, batch operations, and file upload) |
| Error handling | Non-zero exit code ‚Üí `isError: true`, stderr/message as content |
| Guardrails | None ‚Äî the CLI's own confirmation prompts and flags remain the user's responsibility |

## Code Style

- Single file: `src/mcp-server.ts`, target ~100 lines
- No imports from `src/cli/` or any command/config/api modules ‚Äî shell-out only
- Use `child_process.execFile` (not `exec`) ‚Äî safer argument passing, no shell injection
- Resolve binary path via `path.resolve(__dirname, 'index.js')` at runtime (works from `dist/`)
- TypeScript strict mode (matches existing `tsconfig.json`)
- `#!/usr/bin/env node` shebang as first line
- Top-level `await` wrapped in an IIFE or `main()` function
- No `console.log` ‚Äî stderr only for startup messages (`console.error`)

## Boundaries

### Always do
- Shell out to the compiled `dist/index.js` binary ‚Äî never import CLI internals
- Return the CLI's full stdout/stderr verbatim
- Set `isError: true` on non-zero exit codes for `fmo_run`

### Never do
- Add guardrails or command filtering in the MCP layer
- Import from `src/cli/`, `src/api/`, `src/config/`, or any other internal module
- Use `exec` with a shell string (use `execFile` + args array)
- Duplicate any CLI logic in the MCP layer
